# RayleaBot Design Context

## Design Context

### Users
- 个人开发者、自托管机器人部署者、GitHub 开源协作者
- 两种典型使用场景：
  - **巡检模式**：日常打开看一眼状态，快速定位问题，信息密度高、操作路径短
  - **深度配置模式**：配置插件、管理权限策略、设置调度任务，需要清晰的表单和操作引导
- 技术能力强，但对"花哨"有天然免疫力——设计过度会被识别并拒绝

### Brand Personality
- **温暖** — 不是冰冷的命令行工具，是有温度的工作伙伴
- **友好** — 降低自托管门槛，让第一次部署的人也不害怕
- **亲近** — 像一个懂技术的朋友在旁边帮你，不是高高在上的企业软件

### Aesthetic Direction
- **整体基调**：温和的极简主义——克制但不冷漠，精致但不张扬
- **色彩方向**：保留现有蓝色系作为功能色，但将中性色从冷灰向暖灰偏移（微妙的褐底 undertone），让界面在不知不觉中感觉更温暖
- **字体方向**：放弃 Inter 这种"标准答案"无衬线体，引入有温度的显示字体 + 清晰易读的正文字体
- **空间方向**：比典型的开发者工具多一些呼吸感，减少压迫感。卡片用更柔和的圆角和更轻的阴影
- **对话感**：空状态、错误提示、引导文案用"人话"而不是"系统话"
- **不应该做**：不要游戏化、不要拟物化、不要 emoji 泛滥、不要过度拟人。温暖≠幼稚。

### Design Principles
1. **安静服务** — 界面不刷存在感，用户注意力在任务上不在界面上
2. **信息优先** — 巡检场景下数据一目了然，深度场景下路径清晰无歧义
3. **温和存在** — 颜色、间距、动效都往"软"的方向做，不用锋利的分割线和重阴影
4. **人话沟通** — 文案用自然语言，不用"操作成功"这类系统腔
5. **渐进复杂度** — 默认只展示必要信息，需要时展开细节，不吓人

### Technical Constraints
- Vue 3 + TypeScript + Ant Design Vue 4 + Tailwind CSS 4 + Sass
- 必须同时支持亮色/暗色主题（已有 `[data-theme]` 体系）
- 必须支持响应式（已有 1200/960/720/560 断点）
- 必须支持 `prefers-reduced-motion`
- Ant Design Vue 组件可定制 theme token，可通过 `ConfigProvider` 覆盖

### Differentiation
RayleaBot 和同类自托管机器人框架的界面差异点：
- 大多数同类项目界面极简甚至简陋，RayleaBot 提供的是精心设计的管理体验
- 不做"开发者工具就一定得是暗色终端风格"的刻板印象
- 界面让人感到"这是一个被认真对待的产品"，而不是"一个能跑就行"

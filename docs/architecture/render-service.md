# Render Service

本页说明 RayleaBot 当前的图片渲染服务，包括模板来源、版本仓、预览任务、缓存键和管理面能力。

正式接口、错误码和任务结构以 `contracts/` 为准。

## 当前定位

- 图片渲染是平台内建能力，不由插件各自维护浏览器截图链路。
- 当前正式入口包括插件侧 `render.image` 和管理侧 `render.preview`。
- 渲染结果会生成稳定 `artifact_id`，供管理面同源读取和任务详情预览。

## 模板来源与版本真相

- 仓库 `templates/` 提供受控模板基线和首轮种子来源。
- SQLite 保存模板当前版本、历史 revision、最后校验摘要和当前 revision 指针。
- 启动时仅在某个模板尚无已存版本时导入种子模板。
- 内置模板目录继续作为受控基线和缺失诊断依据，运行时当前模板真相来自 SQLite。

## 模板编辑与校验

- 管理面提供模板列表、详情、源码读取、校验、保存、版本历史和回退。
- 源码编辑使用 `manifest_json`、`html`、`stylesheet` 和 `input_schema_json` 四个文件视角。
- 保存和回退都使用 `base_revision_id`，避免静默覆盖当前 revision。
- 校验统一覆盖 JSON 结构、模板 ID、HTML 编译和输入 schema 编译。

## 预览与任务链

- `render.preview` 继续使用现有任务模型和 `/ws/tasks`。
- 管理面预览支持当前模板版本和未保存草稿 `draft.source`。
- 草稿预览只在内存中编译，不写入版本仓。
- 管理面可以在模板页查看任务状态、产物信息、图片结果和任务详情入口。

## 输入与输出

### 输入

- 模板 ID
- 模板输入数据
- 主题与输出格式
- 当前 revision 或未保存草稿源码

### 输出

- 插件侧继续消费 `image_path` 与受控 artifact 信息
- 管理侧通过 artifact 读取接口查看渲染结果
- 失败时返回结构化错误，而不是浏览器原始报错

## 执行模型与缓存

- 渲染任务进入有界队列，由受控 worker 执行。
- 队列长度、并发数、排队超时和执行超时由平台统一控制。
- 缓存键和 artifact identity 包含模板版本、源码摘要、主题、输出格式和输入数据摘要。
- 保存或回退后，即使模板版本号不变，也会因源码摘要变化而生成新的缓存键。
- 渲染失败不会拖垮插件进程；错误摘要进入任务结果、日志和诊断面。

## 资源边界

- Chromium 浏览环境、模板资源与 `.deps/manifest.json` 是当前正式资源来源。
- 模板基线目录和浏览器资源缺失时，平台会在启动、`doctor`、Launcher 和管理面中暴露同一份问题摘要。
- ARM64 或受限环境可通过 `render.browser_path` 显式指定宿主机浏览器路径。
- 运行环境资源可按需下载到 `cache/downloads/runtime/`，并展开到 `.deps/store/<resource-id>/<version>/`。

## 管理面能力

- 系统分组提供 `/render/templates/:templateId?` 模板编辑工作区。
- 页面内显示模板列表、当前版本、最近校验状态、版本历史、输入结构、校验结果和预览结果。
- 本地解析错误、服务端校验错误和预览任务错误会在同一页面内直接展示。

## 当前边界

- 模板编辑器使用轻量源码编辑方式，不引入拖拽式模板搭建器。
- 模板市场、远程发布和更复杂的模板协作能力不在当前正式范围内。
- 高级页面池、浏览器实例复用和更复杂的并发优化保持在后续演进范围。

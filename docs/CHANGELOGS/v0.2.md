# v0.2 Changelog

RayleaBot v0.2 已交付的能力清单与边界。本文件作为历史基线归档。前置基线见 [v0.1.md](./v0.1.md)。

## 主线交付

- OneBot11 完整传输面：`reverse_ws`、`forward_ws`、`http_api`、`webhook` 四条接入链路、协议快照与 transport 异常可见性。
- OneBot11 完整事件与消息兼容矩阵：核心事件、消息段、历史消息与详情读取，覆盖标准、NapCat、幸运莉莉娅三层 provider。
- 在线模板编辑器与可视化：模板列表、源码编辑、schema 校验、任务式预览、保存、版本历史与回退。
- 插件协议扩展：更宽 `action family`、更宽消息段、共享 `error.details` 语义。
- 管理面跨页钻取与诊断引导收口；Web 管理面工程基线固定为 Ant Design Vue + Vue Vben Admin 对齐方案。
- Launcher 状态模型拆分、环境检查收口、WebSocket 事件驱动状态刷新、深链诊断引导。

## Contract / Schema 冻结

- `web-api.openapi.yaml`：OneBot transport、协议快照、日志详情、模板编辑器、治理核心读取面、plugin `display_state` 正式枚举、`PUT /api/config` `apply_effects`。
- `websocket-events.yaml`：协议状态、日志主链与插件生命周期 `display_state` 分支补齐。
- `plugin-info.schema.json`：`default_config`、`role`、`icon`、`repo`、`homepage`、`keywords`、`screenshots`、`platforms`、`system_dependencies`、`concurrency`。
- `plugin-protocol.schema.json`：OneBot 主链所需更宽 `action`、消息段与共享 `error.details`。
- `config.user.schema.json`：OneBot 多 transport 配置主模型、迁移与热更新读取口径。
- `error-codes.yaml`：OneBot transport、compatibility、provider extension、模板编辑相关错误码。
- `release-manifest.schema.json`：`release_manifest.json` 与 `build_info.json` 最小 metadata surface。

## 插件协议与 SDK

- Capability 与 action kind 一一对应；运行时授权对 wider action family、provider action、`message.send` / `message.reply` 统一走 grant 检查。
- Python / Node.js SDK 覆盖正式 OneBot 单动作、3 个 provider 扩展动作、完整消息段 builder。
- 通用回退入口：`onebotAction` / `providerAction`、`onebot_action` / `provider_action`。
- Provider 扩展动作：`provider.napcat.message_emoji.like.set`、`provider.napcat.group.sign.set`、`provider.luckylillia.friend_groups.get`。

## OneBot11 兼容矩阵

- `events`：message / notice / request / meta 完整矩阵、provider 扩展事件。
- `message_segments`：基础、媒体与文件、富文本与卡片、组合与转发、互动、provider 扩展。
- `read_capabilities`：`message.get`、`message.history.get`、`message.forward.get`、`file.get` 等。
- `provider_extensions`：standard / napcat / luckylillia 三层兼容状态固定为 `supported` 或 `unsupported`。
- 管理面：`/protocols/compatibility` 子页展示当前 provider、当前 transport、能力覆盖。

## 模板编辑器与渲染

- `/render/templates/:templateId?` 工作区集成草稿编辑、校验、任务预览、版本回退。
- `base_revision_id` 防止静默覆盖；输入结构、字段说明、必填状态可视化。
- artifact、缓存命中、任务日志和图片结果在同一工作区呈现。

## 管理面与 Launcher

- 协议中心、日志中心、插件、模板编辑器和指令中心之间的稳定标识与工作区 query。
- 日志详情通过 `plugin_id` / `protocol` / `request_id` 提供上下文入口；任务更新通过 `source=tasks` 日志查看。
- Launcher 本地预检收口为安装目录、设置、server 可执行文件、配置与工作目录可写性；运行时资源诊断由服务端裁决。
- Launcher 偏好设置：安装目录主模型、服务端路径与配置文件路径派生、关闭策略三态（`AskEveryTime / HideToTray / ExitApplication`）、托盘菜单。
- 凭据丢失恢复入口（Launcher 偏好页）。

## 发布与门禁

- 发布与自托管 smoke 探测 packaged `/api/protocols/onebot11`、`/api/protocols/onebot11/compatibility`。
- 模板编辑器门禁：列表、源码、校验、预览、artifact、保存、版本历史、回退、重启后 revision 持久化。
- Wider action family：PR 门禁包含 Node / Python SDK 测试、Node SDK `dist` 漂移检查。
- Recovery drill 与长期自托管 smoke 同步验证 v0.2 协议与模板能力。
- `release_manifest.json`、`build_info.json`、`SHA256SUMS.txt` 校验；`onebot_matrix` 作为可选 metadata。

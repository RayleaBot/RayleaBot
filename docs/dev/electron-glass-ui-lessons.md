# Electron 玻璃质感桌面壳实现要点

本文档用于维护 RayleaLauncher 的玻璃质感桌面壳，收敛主进程、渲染层和窗口交互的固定实现约束。

## 主进程约束

- 窗口背景使用 `#00000000`，让系统材质能够透过顶层内容生效。
- 自定义窗口标题栏时保持 `frame: false`，并在需要时启用 `roundedCorners`。
- Windows 平台在 `ready-to-show` 后应用 `setBackgroundMaterial("acrylic")`，保证桌面壳材质和窗口显示时机一致。
- 统一使用深色主题，避免系统浅色标题栏与深色玻璃面板混用。

## 渲染层约束

- `html`、`body`、应用根节点和顶层 Provider 保持透明背景，避免遮挡系统材质。
- 玻璃面板使用低透明度背景、细边框和 `backdrop-filter` 组合，不依赖纯实色面板堆叠层级。
- 需要拖拽窗口的区域设置 `-webkit-app-region: drag`，交互控件显式设置 `no-drag`。
- 顶层布局优先使用稳定的 Grid 或 Flex 结构，保证窗口尺寸变化时标题栏、侧栏和主内容区不互相挤压。

## 窗口交互约束

- 最小化、最大化和关闭动作通过 IPC 暴露给渲染层，避免在渲染层直接假设平台窗口能力。
- 渲染层维护最大化状态订阅，用于同步窗口控制按钮图标和边角表现。
- 托盘恢复和窗口关闭语义保持一致，避免出现标题栏按钮与托盘行为不一致的情况。

## 视觉约束

- 状态徽标、警告条和诊断卡片使用统一语义色，不为单页临时定义独立颜色体系。
- 深色玻璃面板保留足够文字对比度，确保系统状态、错误信息和路径字段在半透明背景上可读。
- 移动或窄窗口宽度下优先保证信息区块纵向堆叠，避免面板横向挤压导致交互控件不可点击。

## 排查清单

- 顶层容器是否仍然保持透明背景。
- 主进程窗口背景是否为 `#00000000`。
- Windows 平台是否在窗口显示前后正确应用 acrylic 材质。
- 标题栏按钮是否都设置为 `no-drag`。
- 最大化状态变化是否能同步到渲染层。
- 面板文本、按钮和状态条在深色玻璃背景下是否保持足够对比度。

# Text Resources

本页说明 RayleaBot 当前文本资源与国际化的正式边界。

## 当前规则

- 平台核心文本使用统一资源键，不把关键提示分散硬编码到多处。
- `zh-CN` 是当前正式默认资源。
- `en-US` 与更多语言保留同一套键结构，作为后续补充语言的扩展方向。

## 使用边界

- Web UI、Launcher、CLI 和平台错误摘要继续围绕统一 `message_key` 组织文案。
- 模板中的固定文案优先使用资源键，而不是把最终文本硬编码进模板文件。
- 多语言扩展不能改变正式错误码、正式字段和正式状态名。

## 当前维护原则

- 文案一致性优先于局部自由发挥。
- 对外显示内容的语义由正式资源键和正式文档裁决，不靠运行时临时拼接。

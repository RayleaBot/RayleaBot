# RayleaLauncher 首页重设计

## 目标

为 `launcher` 启动器首页提供一套稳定、精致、可落地的状态页设计，消除卡片重叠、按钮挤压和中间宽度下的布局失衡，同时保持桌面工具界面的克制信息层级。

## 适用范围

- `launcher/src/renderer/src/AppShell.tsx`
- `launcher/src/renderer/src/style.css`
- `launcher/tests/renderer/style-regressions.test.ts`

本设计仅调整首页 `status` 视图的布局与视觉规则，不改变导航结构、状态文案来源、操作语义、IPC 能力或其他分区的信息架构。

## 设计方向

- 整体基调采用稳重控制台骨架与精致桌面应用质感的组合。
- 首页首屏保持状态信息与操作区均衡，不让任一侧压倒另一侧。
- 视觉层次以稳定、清晰、耐用为先，避免大面积装饰光效与展示型构图。

## 信息结构

首页保持三段式结构：

1. 顶部 `hero` 状态区
2. 中部参数与摘要区
3. 底部日志区

### 顶部 `hero` 状态区

- 左侧显示服务状态、说明文案和错误提示。
- 右侧显示主操作与次操作。
- 状态标题与按钮区权重均衡，任何窗口宽度下都不得出现按钮覆盖标题或提示文案的情况。

### 中部参数与摘要区

- 主列显示核心参数卡。
- 侧列显示版本监控、恢复兼容性和环境预警摘要。
- 侧列信息保持轻量，只承载摘要和单步动作，不与主列争夺视觉重心。

### 底部日志区

- 日志面板独占一整行。
- 日志区保留低亮度深色底板与等宽字体，优先保证可读性。
- 打开日志目录的操作作为日志区附属动作，不进入首页主操作组。

## 布局规则

### `hero` 状态区

- 桌面宽度下使用左右两栏布局。
- 左侧内容采用“状态徽记 + 状态文本 + 说明文案 + 错误横幅”的垂直流。
- 右侧操作区采用固定结构：
  - 第一行 1 个主按钮
  - 第二行 2 个等宽次按钮
- 主按钮必须明显大于次按钮，但不得做成长横幅。
- 次按钮宽度一致，图标和文字水平居中。

### 参数与摘要区

- 桌面宽度下采用主列加侧列布局。
- 主列至少容纳一张完整核心参数卡与一张完整日志卡。
- 侧列宽度使用稳定范围，不依赖单一固定像素宽度。
- 摘要卡统一采用较小标题、较短文案和轻量按钮。

### 卡片系统

- 首页所有卡片复用同一套圆角、边框、内边距与阴影尺度。
- 卡片之间的间距固定，不使用通过外边距硬挤出来的层叠效果。
- 卡片 hover 反馈保持轻量，不使用明显的上浮位移。

## 视觉规则

- 延续当前深色冷调玻璃表面，但降低发光感与堆叠感。
- 首页主视觉依赖留白、排版与按钮比例，不依赖重阴影或大面积高饱和渐变。
- 状态色只用于状态徽记、错误横幅和少量强调文本，不铺满整块卡片。
- 日志区降低背景亮度，避免与上方参数卡产生同级竞争。

## 响应式规则

### 宽屏

- `hero` 维持左右两栏。
- 参数与摘要区维持主列加侧列。

### 中等宽度

- `hero` 操作区下沉到状态内容下方。
- 主按钮保持单独一行。
- 两个次按钮在空间足够时并排，空间不足时整齐改为上下堆叠。
- 摘要侧列下沉到主列后方，整体改为单主列流式布局。

### 窄宽度

- 首页卡片统一单列。
- 长路径、URL 和日志内容保持可换行。
- 不允许遗留固定宽度导致的内容裁切、横向滚动或卡片重叠。

## 组件与样式边界

- 继续使用当前 React + Fluent UI React v9 + 全局 `style.css` 方案。
- 不新增平行组件系统、状态管理方案或样式栈。
- 继续复用既有 `glass-panel`、`panel`、`frost-button` 和状态页语义类；新增样式只用于补齐首页所需的布局与视觉约束。
- 不调整服务状态映射、按钮行为、环境检查分组或日志数据来源。

## 测试要求

- 更新渲染层样式回归测试，覆盖首页中间宽度下的安全布局约束。
- 验证 `hero` 操作区不会出现按钮重叠、标题遮挡或异常换行。
- 验证参数区与侧列在降级为单列时不会保留危险的固定宽度。
- 验证日志区与摘要卡在新布局下仍位于滚动容器内部，不出现裁切阴影或横向溢出。

## 完成判定

- 首页首屏在常见桌面宽度下不存在卡片重叠。
- 主按钮与次按钮的层级明确，布局整齐，交互区稳定。
- 参数卡、摘要卡和日志卡形成统一卡片语言。
- 中等与窄宽度下首页保持可读、可点、无溢出。
- 现有状态页功能保持不变。
